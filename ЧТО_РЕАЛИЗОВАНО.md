# Что реализовано в боте «Битва прогнозов ЧМ-2026»

## Пользовательский функционал

### Регистрация и профиль
- `/start` — регистрация по telegram_id; при повторном вызове обновляет username и имя
- `/me` — личный профиль:
  - место в общем рейтинге
  - очки
  - отставание от следующего места
  - количество точных счётов
  - количество угаданных исходов из всех сыгранных матчей
  - остаток удвоений (из 8)
  - форма последних 5 матчей (🎯✅❌✅🎯, слева-направо от старых к новым)

### Прогнозы
- `/matches` — список предстоящих матчей с фильтрами: сегодня / завтра / неделя / все
- Ввод счёта кнопками (цифры 0–9 за каждую команду)
- Флаг удвоения при вводе прогноза
- Для плей-офф матчей при прогнозе «ничья» — дополнительный вопрос: кто пройдёт дальше (ОТ / ПЕН / не указывать)
- Редактирование прогноза до кикоффа — кнопка «✏️ изменить» в `/matches`
- Блокировка прогноза при попытке поставить/изменить после кикоффа

### Просмотр результатов
- `/my_predictions` — все прогнозы пользователя с результатами и набранными очками
- `/today_results` — результаты матчей за период вчера 15:00 — сегодня 10:00 МСК; по каждому матчу: прогнозы всех участников с иконками и очками
- `/standings` — общий рейтинг с тай-брейком (очки → точные счёты → угаданные исходы)

### Уведомления
- За ~30 минут до матча — напоминание, если прогноз не стоит (одно сообщение, не повторяется)
- После ввода результата — личное сообщение с итогом матча, своим прогнозом и набранными очками
- После последнего матча дня — итог дня: все участники отсортированы по очкам за день
- При открытии нового раунда плей-офф — уведомление всем участникам с призывом поставить

## Административный функционал

### Ввод результатов
- `/result` — пошаговый диалог: список матчей → счёт домашней → счёт гостевой → (если ничья в плей-офф) победитель и способ → подтверждение
- Альтернативный режим: `/result <id> <счёт>` одной командой
- `/playoff_result` — отдельный флоу для записи победителя ОТ/ПЕН к уже завершённому матчу
- `/recalc` — пересчёт очков по матчу (с подтверждением); идемпотентен

### Управление расписанием
- `/add_match` + загрузка CSV — добавление матчей; обязательные колонки: `match_date`, `kickoff_msk`, `stage`, `team_home`, `team_away`

### Статистика и экспорт
- `/check_predictions` — по каждому предстоящему матчу: прогресс-бар ✅⬜ и список тех, кто не поставил
- `/export` — генерация Excel-файла:
  - матчи по датам с прогнозами всех участников
  - ячейки с удвоением выделены оранжевым
  - итоговые строки: по дням (синий), группового этапа (зелёный), общий итог (красный)
  - тонкие границы на всех ячейках
  - заморозка первых 3 строк-заголовков
  - колонка Σ с суммой по строке

### Интеграция с API worldcup26.ir
- `/setup_api` — авторизация (email + password из `.env`), получение и сохранение токена в БД
- `/check_now` — принудительный запуск проверки API без ожидания планировщика
- `/admin_log` — последние 10 действий администратора (result, recalc, add_match) с timestamp

## Движок подсчёта очков

Реализован строго по спецификации:

| Прогноз | Факт | Очки |
|---|---|---|
| Точный счёт | = | 3 |
| Верный исход (победа/ничья) | ≠ счёт | 1 |
| Неверный исход | — | 0 |

**Удвоение:** `base × 2` если base > 0, иначе `−1`. Не распространяется на бонус ОТ/ПЕН.

**Бонус плей-офф** (только если прогноз = ничья):
- Угадал победителя (без метода) → +1
- Угадал победителя + метод (ОТ/ПЕН) → +2
- Не угадал → 0

Функция `score_prediction()` идемпотентна — безопасный повторный запуск через `/recalc`.

**Тай-брейк в `/standings`:** очки → точные счёты → угаданные исходы (всё по убыванию).

## Автоматика (APScheduler)

| Задача | Частота | Что делает |
|---|---|---|
| Блокировка прогнозов | Каждую минуту | Ставит `locked=TRUE` на прогнозы к начавшимся матчам |
| Напоминания | Каждые 5 минут | Ищет матчи через 25–35 мин с `reminder_sent=FALSE`, шлёт тем кто не поставил, ставит флаг |
| Проверка результатов API | Каждые 45 минут | Сравнивает завершённые матчи API с незавершёнными в БД, предлагает подтвердить |
| Проверка новых раундов | Раз в день, 10:00 МСК | Смотрит есть ли в API плей-офф матчи с известными командами которых нет в БД |

При авто-определении ОТ/ПЕН (ничья в плей-офф): бот пытается сам записать победителя если API даёт нужные поля. Если не даёт — присылает кнопку для ручного выбора.

## База данных (SQLite / aiosqlite)

Схема совместима с PostgreSQL (типы, имена) для будущей миграции.

### Таблицы

**`users`** — участники: telegram_id, username, full_name, doublings_left, created_at

**`matches`** — матчи:
- Основные поля: match_date, kickoff_msk, stage, team_home, team_away, status
- Результат: score_home, score_away, went_to_extra_time, ot_pen_winner, ot_pen_method
- Служебные: api_game_id (для дедупликации при импорте из API), reminder_sent (флаг отправки напоминания)

**`predictions`** — прогнозы (UNIQUE на user_id + match_id):
- Прогноз: pred_home, pred_away, is_doubled, playoff_team, playoff_method
- Результат: base_points, base_final, bonus_points, total_points
- Статус: submitted_at, locked

**`audit_log`** — лог действий администратора с JSON payload

**`api_tokens`** — токен доступа к worldcup26.ir

### Виртуальный матч `__adjustment__`

При импорте исторических данных из Excel (групповой этап) неизвестно кто и когда использовал удвоения. Чтобы итоговые очки группового этапа совпадали с оригинальной таблицей, создаётся один специальный матч с `team_home = '__adjustment__'`. В нём хранятся дельты (±N очков) как прогнозы с total_points. Все запросы на реальные матчи фильтруют `WHERE team_home != '__adjustment__'`.

## Импорт исторических данных

`wc26_bet_bot/import_history.py` — одноразовый скрипт:
1. Создаёт 72 матча группового этапа как `finished` с результатами
2. Создаёт 10 пользователей с placeholder telegram_id (1000001–1000010)
3. Импортирует ~718 прогнозов и считает очки
4. Вычисляет дельты удвоений и создаёт `__adjustment__` матч

Реальные telegram_id подставляются вручную через SQL после регистрации участников.
