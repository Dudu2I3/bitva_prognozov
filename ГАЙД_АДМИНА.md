# Битва прогнозов ЧМ-2026 — гайд админа

## Первый запуск

### 1. Создай `.env` файл

В папке `wc26_bet_bot/` создай файл `.env`:

```
BOT_TOKEN=токен_от_BotFather
ADMIN_TELEGRAM_ID=твой_telegram_id
GROUP_CHAT_ID=-100xxxxxxxxxx        # опционально: id группового чата для сводок
WORLDCUP_API_EMAIL=твой@email.com   # опционально: для автополучения результатов
WORLDCUP_API_PASSWORD=пароль
```

**Как узнать свой Telegram ID:** напиши боту `@userinfobot` — он ответит твоим id.

**Как узнать ID группового чата:** добавь бота в группу, перешли любое сообщение из группы боту `@userinfobot` — он покажет id (отрицательное число).

### 2. Установи зависимости

```bash
pip install -r requirements.txt
```

### 3. Запусти бота

```bash
cd wc26_bet_bot
python3 -m bot.main
```

База данных создаётся автоматически в `wc26_bet_bot/data/bot.db` при первом запуске.

## Загрузка расписания матчей

### Вариант А: CSV-файл (рекомендуется для старта)

Отправь боту CSV-файл с колонками:

```
match_date,kickoff_msk,stage,group_name,team_home,team_away
2026-06-11,2026-06-11 21:00:00,group,,Мексика,Польша
2026-06-12,2026-06-12 18:00:00,playoff,,Бразилия,Аргентина
```

- `kickoff_msk` — время строго по МСК, формат `YYYY-MM-DD HH:MM:SS`
- `stage` — `group` или `playoff`
- `group_name` — можно оставить пустым

### Вариант Б: Через API (плей-офф)

Когда на worldcup26.ir появляются матчи нового раунда — бот пришлёт уведомление с кнопкой **«Добавить N матчей»**. Нажимаешь — время автоматически конвертируется в МСК.

Предварительно нужно подключить API командой `/setup_api`.

## Команды администратора

### `/result` — ввод результата матча

Запускает пошаговый диалог:
1. Бот показывает список незавершённых матчей
2. Выбираешь матч
3. Вводишь счёт домашней команды (кнопки 0–9)
4. Вводишь счёт гостевой команды
5. Если плей-офф и ничья → выбираешь кто прошёл и как (ОТ/ПЕН)
6. Подтверждаешь

После подтверждения: очки пересчитываются, каждый участник получает личное уведомление, если последний матч дня — сводка дня.

> Альтернатива: `/result <id матча> <счёт>` — например `/result 42 2:1`
> id матча можно увидеть в `/admin_log` или `/check_predictions`

---

### `/playoff_result` — победитель ОТ/ПЕН для уже введённого матча

Используй если:
- Результат 90 минут уже введён через `/result`
- Но победитель по ОТ/ПЕН не был указан (или API не дал данные)

Бот покажет завершённые матчи без победителя ОТ/ПЕН — выбираешь нужный, указываешь кто прошёл и как. Очки пересчитываются автоматически.

---

### `/recalc` — пересчёт если ввёл неверный результат

Если ошибся в счёте — исправь результат через `/result` (введи тот же матч заново), потом запусти `/recalc` для этого матча. Функция идемпотентна — безопасно запускать многократно.

---

### `/check_predictions` — кто не поставил прогноз

Показывает по каждому предстоящему матчу:
- Прогресс-бар `✅✅⬜⬜⬜`
- Имена тех, кто ещё не поставил

Удобно проверить перед матчем и лично напомнить тому, кто не поставил.

---

### `/export` — таблица в Excel

Генерирует `.xlsx` файл в формате оригинальной таблицы:
- Все матчи по датам
- Прогнозы всех участников
- Ячейки с **удвоением** — оранжевые
- Итоги по дням, группового этапа, финальный итог
- Колонки с цветовой раскраской

---

### `/setup_api` — подключение автоматических результатов

Нужен аккаунт на worldcup26.ir. Логин/пароль прописываются в `.env` (см. выше). После `/setup_api` бот получает токен и сохраняет его в БД.

После этого каждые **45 минут** бот сам проверяет результаты и предлагает их подтвердить. При ничье в плей-офф — пытается автоматически определить победителя по ОТ/ПЕН.

---

### `/check_now` — принудительная проверка API

Запускает проверку результатов прямо сейчас, не дожидаясь планировщика. Полезно после матча если хочешь получить результат быстрее.

---

### `/admin_log` — лог действий

Показывает последние 10 действий: какой матч, какой результат, когда. Полезно если что-то пошло не так и нужно разобраться что вводилось.

## Привязка реальных Telegram-аккаунтов к историческим данным

Если использовался скрипт `import_history.py` для загрузки исторических данных группового этапа — у пользователей стоят заглушки telegram_id (1000001–1000010). Участник не сможет войти в бота пока его реальный ID не будет в базе.

Чтобы привязать реальный аккаунт, выполни SQL напрямую:

```bash
cd wc26_bet_bot
sqlite3 data/bot.db
```

```sql
UPDATE users SET telegram_id = 123456789 WHERE full_name = 'Данияр';
```

Реальный telegram_id участника он может узнать у `@userinfobot`.

После привязки участник пишет боту `/start` — бот обновит его username и имя, всё остальное (прогнозы, очки) уже привязано к `id` в таблице и сохранится.

## Автоматические события

| Что происходит | Когда |
|---|---|
| Прогнозы блокируются | За 0 минут до кикоффа (каждую минуту проверка) |
| Напоминание участникам без прогноза | За ~30 минут до матча (один раз) |
| Проверка результатов через API | Каждые 45 минут |
| Проверка новых раундов плей-офф через API | Раз в день в 10:00 МСК |
| Итог дня — сводка всем участникам | Автоматически после последнего матча дня |

## Частые сценарии

### Ввёл неверный счёт

1. Запусти `/result` — выбери тот же матч — введи правильный счёт
2. Запусти `/recalc` — выбери матч — подтверди пересчёт

### Матч перенесён или отменён

Прямого инструмента нет. Сделай через SQLite:
```sql
UPDATE matches SET kickoff_msk = '2026-07-10 18:00:00', match_date = '2026-07-10'
WHERE team_home = 'Бразилия' AND team_away = 'Аргентина';
```

### Участник пишет «я не могу поставить прогноз»

Скорее всего его telegram_id не привязан к базе (если данные импортировались вручную). См. раздел «Привязка реальных Telegram-аккаунтов» выше.

### Нужно посмотреть базу вручную

```bash
cd wc26_bet_bot
sqlite3 data/bot.db
.tables
SELECT full_name, doublings_left FROM users;
SELECT team_home, team_away, kickoff_msk, status FROM matches ORDER BY kickoff_msk;
.quit
```

### Бэкап базы

Просто скопируй файл `wc26_bet_bot/data/bot.db`. Делай это периодически — всё хранится в одном файле.

## Переезд на сервер

1. Скопируй весь проект на сервер
2. Скопируй `wc26_bet_bot/data/bot.db` если уже есть данные
3. Создай `.env` с теми же переменными
4. Установи зависимости: `pip install -r requirements.txt`
5. Запусти через systemd, screen или supervisor:

```bash
# Простой вариант через screen:
screen -S bitva
cd wc26_bet_bot && python3 -m bot.main
# Ctrl+A, D — отсоединиться
```

> ⚠️ Не запускай бота одновременно на локальной машине и на сервере — два процесса будут конфликтовать за обновления Telegram.
