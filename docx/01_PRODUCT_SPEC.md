# Продукт: Telegram-бот «Битва прогнозов ЧМ-2026»

## Назначение
Заменяет Excel-таблицу для приватного пула прогнозов на ~10 участников. Бот принимает прогнозы на счёт матчей, жёстко блокирует их после начала матча, считает очки по правилам лиги, ведёт рейтинг.

## Участники и доступ
- Доступ только по прямой ссылке на бота — раздаёт админ вручную, только тем, кто оплатил взнос (оплата и распределение призового фонда происходят **вне бота**, полностью вручную)
- Бот не отслеживает оплату и не считает призовой фонд — только финальные места 1/2/3, дальше человек сам применяет 60/30/10%
- Один админ (владелец бота) на v1

## Основной флоу
1. Админ загружает расписание матчей (см. `matches_seed_template.csv`)
2. Участник через `/matches` видит список будущих игр и ставит прогноз
3. Прогноз принимается строго до стартового свистка. После — изменение запрещено; попытка логируется и игнорируется (фактический прогноз остаётся первоначальным)
4. После матча админ вводит результат через `/result`
5. Бот автоматически пересчитывает очки всех участников по этому матчу
6. `/standings`, `/group_standings`, `/final_standings` показывают рейтинг в любой момент

## Команды пользователя
| Команда | Действие |
|---|---|
| `/start` | Регистрация (привязка telegram_id) |
| `/matches` | Список будущих матчей, кнопка "Сделать прогноз" |
| `/me` | Профиль: остаток удвоений (из 8), точность, место в рейтинге |
| `/standings` | Текущий общий рейтинг |
| `/group_standings` | Итог группового этапа (аналог строки 92 в старой таблице) |
| `/final_standings` | Итог всего турнира (аналог строки 141), доступно после финала |

## Команды админа
| Команда | Действие |
|---|---|
| `/result <матч> <счёт>` | Фиксация итога 90 минут, запускает пересчёт очков |
| `/playoff_result <матч> <команда> <ОТ\|ПЕН>` | Фиксация исхода для матчей, завершившихся вничью в плей-офф |
| `/recalc <матч>` | Принудительный пересчёт (если результат поменяли по ошибке) |
| `/add_match` или загрузка CSV | Добавление матчей в расписание |

## Бизнес-правила лиги

### Общие правила
- Ставка — прогноз итогового счёта матча (90 минут)
- Принимается до начала матча
- Изменение после стартового свистка запрещено — любая попытка откатывается к первоначальному прогнозу

### Базовые очки (90 минут)
| Результат прогноза | Очки |
|---|---|
| Точный счёт | 3 |
| Верный исход (победа/ничья), неверный счёт | 1 |
| Неверный исход | 0 |

### Удвоение
- 8 удвоений на участника на весь турнир, использовать можно на любые матчи
- Помечается перед матчем (в боте — флаг/кнопка при отправке прогноза, до дедлайна)
- Если прогноз принёс очки (3 или 1) → очки удваиваются (6 или 2)
- Если прогноз принёс 0 очков → штраф **−1** балл
- Действует **только** на счёт в основное время — не затрагивает бонус ОТ/ПЕН

### Плей-офф: бонус за победителя (ОТ/пенальти)
Доступен только если основной прогноз пользователя на 90 минут — **ничья**.

- Дополнительно к прогнозу счёта можно указать: какая команда пройдёт дальше + опционально способ (ОТ или ПЕН)
- Счёт по ОТ/пенальти не указывается, только победитель (и опционально способ)

| Результат | Очки |
|---|---|
| Угадан победитель, способ не указан | +1 |
| Угадан победитель + верно указан способ (ОТ/ПЕН) | +2 |
| Не угадан победитель ИЛИ указан неверный способ | 0 |

Бонус/штраф удвоения **не распространяется** на исход ОТ/ПЕН — это всегда отдельное, независимое начисление поверх базовых очков (подробный псевдокод — в `02_SCORING_RULES.md`).

### Тай-брейк при равенстве очков в итоговой таблице
1. Больше точных счётов
2. Больше угаданных исходов — победитель/ничья угадан правильно, **независимо от того, угадан ли точный счёт** (т.е. сюда входят и матчи с 3 баллами, и матчи с 1 баллом)

## v1 vs дальше
**v1 (сейчас)**: локальный запуск, SQLite, один админ, ручной ввод результатов матчей.
**Потом (если нужно)**: сервер 24/7, опциональное автополучение результатов через бесплатный API, синхронизация с Google Sheets, несколько админов.
